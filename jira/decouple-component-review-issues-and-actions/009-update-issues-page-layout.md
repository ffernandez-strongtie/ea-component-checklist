# Story: Update Component Review Issues Page Layout

## Value Statement
As a process engineer, I need the Issues page focused on issue maintenance and the selected issue's associated actions so that I can manage issue-specific behavior without the Changes grid competing for space.

## Details
Update the Component Review Issues page after Actions and Changes have been separated into their own pages.

The Issues page should contain:

- Issues grid
- Associated Actions grid for the selected issue
- Manage Actions command for the selected issue

The Changes grid should no longer appear on the Issues page.

## Acceptance Criteria
- Issues page displays the Issues grid.
- Issues page displays an Associated Actions grid.
- Issues page does not display the Changes grid.
- Issues page includes a visible place for the Manage Actions command.
- Layout remains usable at the expected CSE desktop resolution.
- Existing issue fields remain visible unless separately changed by another story.
