# Story: Separate Component Review Actions Administration Page

## Value Statement
As a process engineer, I need a dedicated Actions administration page so that reusable actions can be managed independently from issues.

## Details
Create or update the Component Review navigation so Actions is a separate administration view under Project Settings > Component Review.

The page should show reusable actions and their action-level fields. Because actions are becoming reusable, this page should not present actions as if they belong to a single checklist issue.

This story is focused on page creation, navigation, and display. Add/edit/delete behavior can remain existing behavior or be split into follow-up stories if it cannot be completed in one day.

## Acceptance Criteria
- Component Review navigation includes an Actions item.
- Actions item opens a dedicated Actions administration page.
- Actions page displays reusable actions.
- Actions page displays Action ID, Default, and Job Note fields.
- Actions page does not display Checklist ID as an action owner.
- Existing Administration navigation remains usable.
