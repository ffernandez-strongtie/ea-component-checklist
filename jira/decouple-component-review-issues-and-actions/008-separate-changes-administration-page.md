# Story: Separate Component Review Changes Administration Page

## Value Statement
As a process engineer, I need Changes managed on a dedicated page so that changes are maintained separately from issue-action association work.

## Details
Create or update the Component Review navigation so Changes is a separate administration view under Project Settings > Component Review.

The page should preserve the current Changes and Appended Notes information from the existing combined Component Review screen.

## Acceptance Criteria
- Component Review navigation includes a Changes item.
- Changes item opens a dedicated Changes administration page.
- Changes page displays Changes and Appended Notes fields.
- Existing change records remain visible.
- Moving Changes out of the Issues page does not remove or alter existing change data.
- Existing Administration navigation remains usable.
